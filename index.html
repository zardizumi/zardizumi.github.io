git

Git入门

1.在一个文件夹中打开git bash使用git init命令 初始化一个项目仓库,会生成一个.git文件夹,默认隐藏
2.直接在这个夹中写代码

Git设置用户信息

设置用户信息后,备份之后会存储备份者信息

将代码存储到.git暂存区然后在存进版本库

使用git status查看当前项目目录中的代码处于哪个区域
流程 工作区(git status 红色)=>暂存区(git status 绿色)=>版本库 (nothing to commit, working directory clean)

$ git status 查看当前项目目录中代码是否被放到仓库中

.gitignore 忽略文件

被忽略的文件不会被git status 显示出来,也就不会被提交
新建.gitignore文件,写法 /文件路径,一行写一个

删除版本库中的代码

直接在工作区删除代码,再次提交,这样就删除了这个代码,但是之前的可以恢复

查看历史日志 git log

git 版本回退

也可以可以使用git log查看版本号,通过git reset –hard xxxx 来精确回退到某个版本
回退之后,只会显示以当前版本为基准之前的日志,之后的不会显示.

版本号切换

使用git reflog查看详细记录,可以查看到以前的版本号
通过git reset –hard 版本号 切换到任意版本

创建,切换,合并分支

##处理合并时候的冲突 ##

先在master提交了文件,然后进入分支,修改同一个文件,再次提交,然后回到master,合并就会出现矛盾. 解决矛盾需要手动删除矛盾的部分,

将代码上传到github

从github拉取代码

1.初始化一个git仓储
2.通过git pull 地址 master 拉取代码

或者直接使用git clone 地址 他会新建一个文件夹

ssh上传

通过ssh-keygen -t rsa -C ‘邮箱地址’ 生成公钥和私钥
在文件夹中找到.pub结尾的是公钥，将公钥中的内容复制到github设置。

事件委托

JavaScript中的事件委托

应用场景

同时为父元素下所有的子元素添加事件,如果不使用事件委托,则需要为每一个元素单独添加事件.
如果使用事件委托则只需要为父元素绑定事件,即可让每一个子元素都绑定上响应的事件
