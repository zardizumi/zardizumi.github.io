ES6笔记

新的变量声明方式

var

通过var在全局作用域声明全局变量会污染全局环境

let

let声明的变量只有先声明再使用否则会报错.
for循环中使用let声明i,这个i不会被释放到循环体外中,不会污染全局环境.

const

使用const声明的是常量,被声明后不能再被赋值.

比如可以在项目中用const作为引入的插件或库的声明,防止之后再被改变

总结

以后尝试用let代替var来

变量的解构赋值

ES6允许按照一定模式从数组和对象中提取值对变量进行赋值

数组解构赋值

左边的解构模式必须与右边的解构模式一样.

设置解构的默认值

undefined 不能代替默认值,而null可以替代默认值

对象的解构赋值

数组的解构赋值只要顺序对就可以赋值成功,而对象的解构赋值是找对应的变量名,只有找到对应的变量名才能赋值成功

用let或var先赋值,在进行解构必须在解构之外加上圆括号才能结构成功

字符串的解构赋值

扩展运算符和rest运算符

对象扩展运算符’…’

当我们不确定方法传入参数的个数时

使用对象扩展运算符传参数,即使参数数量不正确也不会报错,而ES5中则会报错.

使用扩展运算符对数组进行赋值
在ES5中如果等于号使用对数组进行赋值,则是将arr1的内存地址映射给arr2,如果arr1改变,arr2一样会改变.
而在ES6中如果使用…对数进行赋值,则只是将arr1的值arr2,等于创建了一个新的arr2数组.


rest运算符(剩余部分) ‘…’

rest和扩展运算符都是’…’用法几乎相同,但是rest有剩余的意思,它只会算通过…传入的参数部分,通过直接传入的不会算进去


使用for of循环打印数组

使用for of替换for 循环可以避免在内存中开辟空间来存索引i

字符串模板

ES5中使用’+’来拼接字符串,而在 ES6中要使用`${字符串}`和来配合,这就是字符串模板并且可以在里面嵌入html标签

字符串模板也支持运算


字符串查找 被搜索字符串.includes(要搜索字符串)
查找开头有没有 被搜索.startsWith(要搜索)
结尾有没有被搜索.endsWith(要搜索)
字符串复制 要被复制.repeat(次数)

Proxy

代理,是对对象和方法的增强,在使用或操作函数和对象的特定时候,进行预处理.类似钩子函数.
Proxy有两个参数第一个参数预处理对象,第二个是包含处理函数的对象,这个对象要包含三个值 get set apply,其中get和set针对对象,apply针对函数
如果想要真正的拿到值,必须使用return target[key] 才能将对象的值真正拿到
格式:

target表示该对象,key表示被改变的键名,value表示改变后的值,receiver表示Proxy对象.

声明proxy

target是要进行预处理的对象或函数,handler是预处理函数

get

在拿到对象值之前进行处理

set

拿到值之前对值进行处理

所以执行了一次值的改变,再输出这个值的过程,会依次输出here is proxy ;将luan赋值为james;james

apply

apply是针对函数的预处理
固定语法,在apply中return Reflect.apply(…arguments)

Promise

Promise承诺,可以理解为当成功的时候执行一些代码,当失败的时候执行一些代码,可以避免过多的回调嵌套.

声明一个执行状态

声明三个执行函数

resolve表示当成功的时候执行的代码,函数,等等,reject表示当失败的时候执行的代码函数

创建promise实例

new Promise(起始函数).then() 有几个步骤就可以写几个then,then里可以传一个参数,这个参数则是失败或成功时执行的代码

总结

1.Promise是一个对象,该对象里面保存着未来才会结束的事件(通常是一个异步操作)的结果,这个对象可以传两个参数一个是resolve表示从正在执行切换成已成功,reject表示从正在执行切换成已失败.
2.想使用Promise只需要在一个函数中返回 new Promise(resolve,reject),在执行完函数体内的函数,就会执行resolve或reject
3. resove或reject后可以通过then分别指定成功和失败后执行的函数
4.then中还可以继续返回new Promise 继续进行上述过程.
5. 步骤: new Promise=>成功(resolve)or失败(reject)=>then(成功回调,失败回调),并可以继续返回Promise对象 =>…….
