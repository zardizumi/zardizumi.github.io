vue虾米实战

  • Vue


一个高仿虾米网页端!
项目地址:http://www.zardluan.cn/xiami/

前言

自学了vue大概一个星期,突发奇想,想模仿一个世面上的的SPA试试看,找来找去,发现虾米音乐的这个不算太难,所以准备动手试试.
使用技术:webpack,vue-cli,vue-router,可能会用到vuex,因为是模仿,可能不需要维护那么多状态.

一.页面布局

整体页面固定的部分是头部的tab栏,和底部的mini播放器,之前没有考虑到底部mini播放器要在别的页面公用,本来应该做成组件的.结果我直接将其新歌页面的模板的一部分了.

业务逻辑部分

1.首先我希望被点击那一个歌曲被激活,作为激活的歌曲,所以我声明了一个activeMusic变量,这个变量是一个对象,对象中存着激活的歌曲的歌名,歌手和专辑图片

2.其次,当该页面被创建之后,我希望mini播放器上有激活歌曲的专辑图片,和歌手名,歌名.而这个激活歌曲是当前音乐数据的第一组数据

3.最后当我点击一个歌曲之后将点击的那个对应歌曲的数据传入mini播放器.

4.点击播放功能

组件化Mini播放器,并且添加使用vuex进行状态管理

1.逻辑部分
1.先构建一个mini播放器组件,插入根节点.
2.mini播放器中需要使用共享状态的地方,打开页面后将歌曲列表第一首加入激活歌曲列表.并且当在页面上点击一首歌,会用这首歌替换掉当前激活列表的歌曲.

2.实现
1.在项目目录对vuex进行安装 npm install vuex --save
2.在项目目录src下新建vuex文件夹,在该文件夹下新建store.js
3.在store.js中引入vue,vuex,并在Vue中应用vuex
import Vue from 'vue';import Vuex from 'Vuex';Vue.use(Vuex);

4.在store.js下声明要共享的状态newmusicData,newactiveMusic.然后将state暴露出去

5.在mini播放器组件中.引入store.又因为要使用多个状态,所以引入{mapState}

6.我们在组件被挂载后给用newmusicData状态的第一个元素给newactiveMusic赋值.达成打开即播放的功能.

7.在新歌页面,修改歌曲的点击事件,将点击歌曲的属性赋值给newactiveMusic.

四.排行榜,歌单,歌手页面

这几个页面的本质都是相同的,都是先从服务器拉取数据,再渲染到页面上.
同时使用vuex来管理需要被多个组件或页面使用的数据

其中歌手页面的逻辑:进入页面=>从服务器拉取数据=>渲染大分类=>单击其中一个分类就跳转路由进入展示该分类详细信息的页面=>在store.js中声明currentArtist状态用于管理当前点击的详细歌手数据=>在新的页面中使用currentArtist来渲染歌手的详细信息
