  • vue

关于Vue中组件之间的通信


父子组件通信

Vue中,组件与组件之间的作用域是孤立的,意味着,我们不能在直接在子组件中引用父组件的数据,所以vue中父子组件通信宗旨:父向子下发数据,子通过事件告知父一些信息

父组件通过props向子组件下发数据

子组件必须在内部声明prop属性,这个属性的值是一个数组,保存着它要引用的父组件的数据
而父组件通过 :message=’数据名’ 或message=’数据内容’的形式给子组件下发数据

直接绑定父组件的对象

我们可以给子组件绑定一个不带任何参数的v-bind,将对象传入,这样就相当于一次性将这个对象的每一条属性分别绑定到子组件上

字符串和动态语法

通过 message=’1’这种方式绑定的仅仅是一个字符串,如果想让子组件的数据随父组件数据改变而动态改变则需要使用:message=’父组件的数据’来绑定

单向数据流

通过props给子组件下发数据应该是单向的,即父组件数据发生更新,将传导给子组件
而我们不应该直接修改子组件的数据.

修改子组件中数据的方法
1.在子组件中定义一个局部变量,初始化这个数据

2.在子组件的计算属性中处理数据并返回

let app = new Vue({
    el: '#app',
    data: {
       msg:'hellow world'
    },
    components:{
       'chen':{
           template:`<h4>{{messages2}}</h4>`,
           props:['message'],
           //对子组件传过来的数据进行处理,并返回一个新的数据
           computed:{
              messages2:function() {
                   return this.message+'haha';
              }
           }
       }
    }
})

替换合并特性

如果在组件上使用了bootstrap的某些功能,那么bootstrap会为组件自动添加一些类名,这是如果我们也为其另外添加了类名,则这两组类名会自动合并.
具有同样功能的还有style

子组件与父组件通信

子组件触发父组件的自定义事件来与父组件通信

子组件通过在内部通过$emit(‘自定义事件名’)触发自定义事件来与父组件通信,即,父组件通过v-on:自定义事件来监听事件,子组件通过$emit()来触发自定义事件与父组件通信

为组件绑定原生事件

如果想为组件绑定一个定义在根组件的事件,那么要通过 v-on:事件.native来绑定

使用插槽分发内容

什么叫插槽,什么叫分发内容

插槽就是slot标签,分发内容是指在父组件中使用子组件的情况下,在父组件模板中将要分发的内容写在子组件两个标签之内,然后再在子组件模板中写上一对slot标签作为插槽,分发的内容就会显示在插槽区域内

一句话:子组件模板中写插槽slot,父组件中分发内容

普通内容分发

具名插槽

每一个slot标签可以有一个name属性来指引内容的分法,即为具名插槽.
具有name的插槽将匹配内容中带有相应slot特性的元素.
不具有name属性的为默认插槽,匹配不到的内容将被插入这里.
