Redux

Redux总体思想

Store

Store可以理解为一个仓库,用于存储状态,一个应用应当只有一个Store

State

State包含所有的数据.可以通过store.getState()方法获取当前时刻的state

Action

改变state的唯一途径就是通过Action.Action 描述当前发生的事情。改变 State 的唯一办法，就是使用 Action。它会运送数据到 Store。

Action Creator

Action Creator用于快速创建action,当需要发送很多不同的信息时可以简便一些

store.dispatch(action)

store.dispatch()是view发送action的唯一方法

Reducer

Reducer是store接收到actions后对当前state进行操作,然后返回一个新状态的函数

实际开发中使用store.dispatch()方法会自动触发reducer,所以在创建store的时候要将reducer作为参数传进去
