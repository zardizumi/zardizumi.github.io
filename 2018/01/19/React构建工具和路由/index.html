React构建工具和路由

  • React

react-create-app

项目目录

项目入口文件是src/index.js,根组件是src/App.js,基础模板文件在public/index.html

自主构建generator-react-webpack

generator-react-webpack依赖于yeoman所以要先全局安装yeoman,再全局安装generator-react-webpack

使用yeoman构建的react项目目录结构

进入项目目录,使用 yo react-webpack 可以使用yeoman快速构建react项目
``

1.package.json:依赖项,以及支持我们使用的命令


2.项目入口在src/index.js


3.页面入口src/index.html


4.根组件在src/components/Main.js

手动使用webpack构建项目

1.使用npm init 初始化文件夹
2.安装webpack npm i –save-dev webpack

配置入口和出口文件

1.在项目目录下新建webpack.config.js文件,在里面配置出入口文件

2.项目目录下新建index.html,引入dist/index.js
3.配置package.json,中的scripts属性,将打包命令配置进去 “build:”webpack”
3.使用npm build打包,打开index.html预览效果

配置开发服务器和热更新

1.在项目目录安装webpack-dev-server npm install –save-dev webpack-dev-server
2.在webpack.config.js中配置devSever属性,注意可以使用ipconfig来查看本机ip,将host设置为本机ip

3.在package.json配置启动服务的脚本, “start”:”webpack-dev-server –open”,启动命令自动打开浏览器预览
4.此时还不能在更改文件后自动刷新,需要配置出口文件的publicPath属性,将其设置为temp/缓存目录

完整配置文件

配置React和Babel

配置Babel是为了能让项目支持ES6和React
1.安装四个Babel包,cnpm install –save-dev babel-core babel-preset-es2015 babel-preset-react babel-loader
2.在webpack.config.js中配置module,配置loaders

3.安装React和ReactDOM
cnpm i –save react react-dom
4.在index.js文件中写组件并挂载到index.html中

完整代码

路由的exact属性 精确匹配

在路由Route 或Link组件上加上exact属性用于对路径进行精确匹配,比如如果路由路径分别是’/‘ 和 ‘/a’如果不加exact精确进行精确匹配,那么就会同时匹配/ 和/a 两个路由因为两个路径都有斜杠,则页面会显示两个页面.如果加上exact属性,则该路由会精确匹配/ 或/a

引入css文件 className 属性

1.写style.css文件
2.因为默认webpack是支持css文件的,所以要配置loader
3.安装style-loader和css-loader
4.在webpack.config.js中配置loader

4.直接在组件的js中通过import ‘./style.css’导入样式表
5.为组件加类名必须使用className

activeClassName属性 设置路由激活状态

在路由连接上加上activeClassName属性,给其赋予类名,则可以让该链接在被激活时添加类名

404页面的设置 Switch组件的使用

1.如果要设置404页面,需要现有一个当路由不存在时跳转的组件

2.在index.js中从react-router-dom中导入Switch组件,Switch组件的作用是进行路由匹配,我们需要将当全部路径都匹配不上时的路由放在最后.并且不给它设置to属性,设置其component属性为404组件

Redirect组件 重定向

Redirect组件主要用于当从一个不存在路由要跳转到一个存在的路由
1.在链接组件中新建一个不存在的链接

2.在index.js中从react-router-dom中导入Redirect组件,并在根组件中加入Redirect组件,设置其from属性,from为从哪里跳转而来,设置其to属性,to为跳转到哪里

通过路由传值

通过路由传值并不是最好的方式
1.在中通过to将传的值绑定进去
2.在Link组件中将值传进去
3.在Two组件中通过this.props.match.params接收传递的值,并将值渲染到页面中
4.传多个值的情况下,多个值直接用/区分,并在路由设置中再添加/:param区分

Router(路由器)的属性

basename (string)
可以在路由中加一层级,比如/one 变成/demo/one 只需要设置basename=’demo’

forceRefresh (boolean)
当你为Router标签设置这个属性为true后,路由将不起作用,一般用处,用于第一次从后台加载页面,路由启用,之后直接从本地缓存加载,不需要路由,减少对后台的请求.

路由模式

路由模式分为BrowserRouter,HashRouter,

HashRouter

会在路由前加上#,不会因为刷新页面而找不到路径

MemoryRouter

不存储history，所有路由过程保存在内存里，不能进行前进后退，因为地址栏没有发生任何变化。

NativeRouter

经常配合ReactNative使用，多用于移动端，以后ReactNative课程中会详细讲解。

staticRouter

设置静态路由，需要和后台服务器配合设置，比如设置服务端渲染时使用。

Prompt组件

当离开页面时会弹出一个框提示是否离开

message属性

提示信息

when 属性

是否启用 接收boolean值

官方实例总结

三个组件

Square组件

Square组件是棋盘上的每一个方格,因为如果在该组件上声明状态那么这个状态就被每一个组件给私有了,所以属性要统一由父组件传过来
这个组件接收两个属性,一个是由其父组件,Border棋盘传过来的value属性,另一个是棋盘传过来一个函数

Border组件

1.该组件声明了一个状态squares,是所有方格的数组用于管理所有方格的状态.以及renderSquare方法,用于渲染方格和传递状态,handleClick方法则为点击方格之后触发的处理函数.
2.属性传值,有Border传递给每一个方格Square,

函数定义组件

因为我们的Square组件已经不具备constructor了,所以我们可以直接用一个函数定义它,传入props,去掉所有的this,只需要返回一个JSX元素即可并在棋盘组件中声明

轮流落子

轮流落子的原理就是声明了一个标记状态,默认下一个落子是X,xTurn:false,每次点击判断xTurn的值,并且改变其值
